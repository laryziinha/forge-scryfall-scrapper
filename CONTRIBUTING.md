## Contributing code (Pull Requests)

If you want to submit code:

1. Fork the repository
2. Create a branch for your change
3. Make your changes with clarity and minimal scope
4. Open a Pull Request using the provided template

### Guidelines
- Keep PRs **focused** (one fix or feature per PR)
- Avoid large refactors unless discussed first
- Follow existing naming and folder conventions
- Test your changes locally when possible

PRs without a clear description or context may be closed without review.

---

## Project scope

This project focuses on:
- Forge-friendly image downloads
- Scryfall API integration
- Practical handling of sets, tokens, singles, and audits

Out-of-scope requests (UI frameworks, unrelated tools, heavy redesigns) are
unlikely to be accepted.
